Capture guidance:

1. Start the backend API, Redis, worker, and frontend.
2. Use a prepared trip with itinerary, places, checklist, budget, and share data.
3. Capture desktop screenshots at roughly `1440x1000`.
4. Capture mobile workspace at roughly `390x844`.
5. Avoid exposing private email addresses, real booking references, API keys, tokens, or personal travel details.
6. Prefer populated demo data over empty states.

Suggested sequence:

1. Landing page before sign-in.
2. Dashboard after sign-in.
3. Workspace itinerary view.
4. Chat with a useful planning response.
5. Build My Trip while queued/running or showing recent activity events.
6. Regenerate-day dialog or completed regeneration activity.
7. Public read-only share page.
8. Mobile workspace itinerary tab.
